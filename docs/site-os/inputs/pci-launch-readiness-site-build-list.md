# Pest Control Inc. Launch Readiness Site Build List

## Purpose

This document reorganizes the Pest Control Inc. website architecture into a practical launch-readiness build order.

The original architecture organizes the website by page type and SEO cluster. This version reorganizes the same site plan by what should be built first, second, third, and ongoing based on:

- Lead generation readiness
- Conversion value
- Parent page authority
- Local search value
- Service demand
- Crawl and index structure
- Internal linking readiness
- Schema readiness
- Ability to publish pages without creating thin or duplicate content

## Important Architecture Update

All approved services and all approved locations will have a dedicated service + location page.

This changes the earlier limited service + location plan into a full matrix.

### Previous model

A small set of service + location combo pages.

### Updated model

Every service gets a page for every approved location.

Example:

- Scorpion Exterminator Henderson NV
- Rodent Exterminator Summerlin
- Ant Exterminator Green Valley
- Cockroach Exterminator North Las Vegas
- Termite Exterminator Boulder City NV
- Bed Bug Exterminator Spring Valley NV

## Recommended URL Slug Rule

Use a flat, keyword-first URL structure for service + location pages.

### Primary pattern

`/{service-slug}-{location-slug}/`

### Examples

| Page | URL Slug |
|---|---|
| Scorpion Exterminator Henderson NV | `/scorpion-exterminator-henderson-nv/` |
| Ant Exterminator Summerlin | `/ant-exterminator-summerlin/` |
| Rodent Exterminator North Las Vegas | `/rodent-exterminator-north-las-vegas/` |
| Cockroach Exterminator Green Valley | `/cockroach-exterminator-green-valley/` |
| Termite Exterminator Boulder City NV | `/termite-exterminator-boulder-city-nv/` |
| Bed Bug Exterminator Spring Valley NV | `/bed-bug-exterminator-spring-valley-nv/` |

## Slug Rules

- Service keyword comes first.
- Location modifier comes second.
- Use lowercase letters.
- Use hyphens only.
- Avoid dates.
- Avoid unnecessary stop words.
- Use one canonical location name per page.
- Avoid publishing overlapping duplicate pages unless there is clear search intent difference.
- Use parent service pages and parent location pages to support internal linking.
